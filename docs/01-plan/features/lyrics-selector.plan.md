# Plan: 가사 후보 선택 기능 (Lyrics Selector)

> 가사 매칭이 안 맞을 때 다른 가사 후보를 선택할 수 있는 기능 + Netease 멀티 소스

---

## Executive Summary

| 관점 | 설명 |
|------|------|
| **Problem** | LRCLIB 자동 매칭이 틀린 가사를 가져오거나, 한국 음악에서 가사를 못 찾는 경우가 있으나 수동으로 바꿀 방법이 없음 |
| **Solution** | 가사 팝업에 "다른 가사" 버튼 추가, LRCLIB + Netease 멀티 소스 후보 목록을 표시하여 사용자가 선택 |
| **기능/UX 효과** | 가사 매칭 실패 시 원클릭으로 올바른 가사로 전환 가능, 한국 음악 커버리지 대폭 향상 |
| **Core Value** | 멀티 소스 가사 검색 + 사용자 선택으로 가사 정확도 극대화 |

---

## 1. Background & Problem

### 현재 상태
- `fetchLyrics()`가 LRCLIB API에서 6단계 검색 후 자동으로 "best" 하나만 선택
- `lrclibSearch()` 내부에서 한글 > 비일본어 > 싱크 우선순위로 자동 필터링
- LRCLIB은 영어 중심 DB (약 300만곡)로 한국 음악 커버리지가 부족
- 사용자가 가사를 변경할 수 있는 UI가 없음

### 문제점
- 동명이곡, 리메이크, 커버곡 등에서 잘못된 가사가 매칭됨
- 일본어/한국어 판별 로직이 원치 않는 버전을 선택하는 경우
- LRCLIB에서 한국 음악 가사를 못 찾는 경우가 빈번
- 사용자가 할 수 있는 건 싱크 타이밍 조절뿐, 가사 자체를 바꿀 수 없음

---

## 2. Goal & Scope

### Goal
가사 팝업에서 LRCLIB + Netease 후보 목록을 보고, 원하는 가사를 선택할 수 있게 함

### In Scope
1. 가사 탭 하단에 "다른 가사" 버튼 추가
2. 버튼 클릭 시 LRCLIB + Netease 후보 목록 표시 (트랙명, 아티스트명, 싱크 여부, 소스)
3. 후보 선택 시 해당 가사로 교체
4. 선택한 가사를 곡별로 기억 (다음 재생 시 동일 가사 사용)
5. Netease (网易云音乐) 가사 소스 추가 — 한국 음악 커버리지 보강
6. 자동 검색 시 LRCLIB → Netease 폴백
7. Netease 중국어 전용 가사 필터링

### Out of Scope
- 사용자가 직접 가사를 입력/편집하는 기능
- 가사 검색 키워드를 직접 변경하는 기능
- Musixmatch 등 유료 API 연동

## 3. Requirements

### 3.1 후보 목록 조회 (P0 - Must)

| ID | 요구사항 | 상세 |
|----|----------|------|
| R-01 | 후보 목록 API | LRCLIB + Netease 검색 결과를 반환하는 IPC 추가 |
| R-02 | 후보 정보 표시 | 트랙명, 아티스트명, 싱크 유무, 가사 미리보기, 소스(LRCLIB/Netease) |
| R-03 | 현재 가사 표시 | 현재 선택된 가사에 표시 |

### 3.2 가사 선택 UI (P0 - Must)

| ID | 요구사항 | 상세 |
|----|----------|------|
| R-04 | "다른 가사" 버튼 | 가사 탭 sync-bar 내 배치 |
| R-05 | 후보 목록 오버레이 | 가사 영역 위에 풀스크린 목록 표시, 스크롤 가능 |
| R-06 | 후보 선택 | 클릭 시 해당 가사로 즉시 교체 |
| R-07 | 목록 닫기 | 선택 후 자동 닫기 또는 X 버튼으로 닫기 |

### 3.3 선택 기억 (P1 - Should)

| ID | 요구사항 | 상세 |
|----|----------|------|
| R-08 | 곡별 가사 선택 저장 | videoId별로 선택한 소스+ID 저장 (`lyrics-selection.json`) |
| R-09 | 재생 시 자동 적용 | 저장된 선택이 있으면 해당 가사 우선 사용 |

### 3.4 Netease 멀티 소스 (P0 - Must)

| ID | 요구사항 | 상세 |
|----|----------|------|
| R-10 | Netease 검색 | `music.163.com/api/search/get/` 검색 |
| R-11 | Netease 가사 조회 | `music.163.com/api/song/lyric` LRC 파싱 |
| R-12 | 자동 폴백 | LRCLIB 실패 시 Netease 자동 검색 |
| R-13 | 중국어 필터링 | 중국어만 있고 한국어가 없는 가사 제외 |

## 4. Technical Approach

### 4.1 변경 파일 목록

| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| `main.js` | 수정 | Netease API 함수, 후보 IPC, 선택 저장/로드, 폴백 로직 |
| `preload-ui.js` | 수정 | 새 IPC 채널 expose (2개) |
| `renderer/mini-lyrics-popup.html` | 수정 | "다른 가사" 버튼 + 후보 목록 오버레이 + 소스 뱃지 |

### 4.2 가사 검색 흐름

```
[곡 재생]
  1. 저장된 선택 확인 (lyrics-selection.json)
  2. LRCLIB 검색 (기존 6단계)
  3. LRCLIB 실패 시 → Netease 폴백

["다른 가사" 클릭]
  LRCLIB 후보 (최대 10개) + Netease 후보 (최대 8개) 동시 표시
  → 중국어 전용 가사 자동 필터링
  → 사용자 선택 → 즉시 교체 + 곡별 기억
```

## 5. Risk & Mitigation

| 리스크 | 영향 | 대응 |
|--------|------|------|
| LRCLIB + Netease 모두 결과 0개 | 후보 목록이 비어있음 | "다른 가사를 찾을 수 없습니다" 메시지 |
| Netease 중국어 가사 혼입 | 사용자 혼란 | 중국어+한국어 없음 → 자동 필터링 |
| Netease API 불안정/차단 | 폴백 실패 | LRCLIB 우선, Netease는 보조 소스 |
| 후보 검색 시간 | 두 소스 동시 검색으로 지연 | "검색 중..." 로딩 표시 |
| 선택 저장 파일 비대화 | 디스크 사용 | 최대 500곡 제한 |
